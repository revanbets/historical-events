SECTION 1 — QUICK SYSTEM MAP (What's Running)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  5 external services power the database:

  1. GITHUB      — Stores the code. The "master copy" of the project.
  2. NETLIFY      — Puts the website online. Reads from GitHub automatically.
  3. SUPABASE     — The database. Stores all events, research, and user data.
  4. RENDER       — The AI brain server. Runs the Python backend (analysis,
                    video transcription, file reading).
  5. ANTHROPIC    — The AI company (makes Claude). Used for document & video
                    analysis. You pay per use.

  MOBILE APP (local — not a hosted service):
  6. NODE.JS      — Software installed on your Mac that lets you run the mobile
                    app development server. Free, one-time install.
  7. EXPO GO      — Free app installed on iPhones/Android phones. Lets anyone
                    preview the mobile app without going through the App Store.

free. No accounts or API keys needed.

  • yt-dlp             — Downloads or streams YouTube videos. Used to grab
                         captions and video streams for frame extraction.
  • OpenAI Whisper     — AI transcription tool. Converts video/audio to text
                         when no captions are available. Runs entirely on the
                         Render server (no OpenAI account needed).
  • OpenCV             — Extracts screenshots (frames) from videos.
  • pdfplumber         — Extracts text from PDF files.
  • python-docx        — Extracts text from Word documents.
  • pytesseract        — Reads text from images (OCR).
  • Trafilatura /      — Scrapes text content from web pages.

Do this once a month to keep everything healthy:

  □ Check Anthropic usage + spending
      → console.anthropic.com → Usage
      → Make sure you have credits and haven't hit your limit

  □ Check Netlify bandwidth
      → app.netlify.com → your site → Analytics
      → Stays well under 100 GB unless you have thousands of daily users

  □ Check Supabase database size
      → supabase.com/dashboard → your project → Database
      → 500 MB free. Text data is tiny — you'd need millions of events to
        approach this limit.

  □ Check Render logs if AI analysis has been acting up
      → dashboard.render.com → your service → Logs tab

  □ If Supabase paused (site loads but no data appears)
      → supabase.com/dashboard → your project → click "Restore project"

SECTION 8 — QUICK TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  PROBLEM → LIKELY CAUSE → WHERE TO CHECK

  Site is down / won't load
    → Netlify outage or deploy failed
    → app.netlify.com → look for failed deploy in red

  Events not loading / blank database
    → Supabase paused (inactive for 7+ days)
    → supabase.com/dashboard → restore the project

  AI analysis not working / "Analysis failed" error
    → Could be: Render asleep (wait 30 sec and retry)
    → Could be: Anthropic API key expired or out of credits
    → Check: console.anthropic.com → API Keys + Usage

  First upload after a while takes forever (~30 seconds)
    → Normal. Render (the backend server) is waking up from sleep.
    → Nothing is broken. Retry the upload once it's awake.

  USER GUIDE:   /Users/mac/Desktop/Claude research app/mobile-app/USER_GUIDE.txt


┌─────────────────────────────────────────────────────────────────────────────┐
│  PREREQUISITE — Node.js must be installed on your Mac                       │
├─────────────────────────────────────────────────────────────────────────────┤
│  Node.js is free software that lets you run the mobile app development      │
│  server. You only install this once.                                         │
│                                                                             │
│  INSTALL:  Go to https://nodejs.org → download "LTS" version → install it  │
│  VERIFY:   Open Terminal, type:  node --version                              │
│            Should print something like v20.x.x                              │
│                                                                             │
│  Without Node.js, the next steps will not work.                             │
└─────────────────────────────────────────────────────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────────┐
│  HOW TO START THE MOBILE APP (Development Server)                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  FIRST TIME ONLY — install dependencies (do this once after setup):        │
│    1. Open Terminal                                                          │
│    2. Type exactly:                                                          │
│         cd "/Users/mac/Desktop/Claude research app/mobile-app"              │
│         npm install                                                          │
│    3. Wait for it to finish (1-2 minutes)                                   │
│                                                                             │
│  EVERY TIME — start the development server:                                 │
│    1. Open Terminal                                                          │
│    2. Type:                                                                  │
│         cd "/Users/mac/Desktop/Claude research app/mobile-app"              │
│         npx expo start                                                       │
│    3. A QR code appears in the Terminal                                      │
│                                                                             │
│  ON YOUR PHONE:                                                             │
│    • iPhone: Open Camera app → point at QR code → tap the notification     │
│    • Android: Open the Expo Go app → tap "Scan QR code"                    │
│                                                                             │
│  RESULT: The app opens on your phone. Any changes you make to the code     │
│    refresh automatically on the phone.                                      │
│                                                                             │
│  TO STOP: Press Ctrl+C in the Terminal window                               │
└─────────────────────────────────────────────────────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────────┐
│  SHARING WITH FRIENDS & FAMILY (No App Store Required)                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  OPTION A — Same WiFi Network (Instant, easiest)                           │
│    • Start the app server: npx expo start                                   │
│    • Friends install "Expo Go" from the App Store (free)                   │
│    • They scan your QR code — app opens immediately                         │
│    • Limitation: only works when you and friends are on the same WiFi      │
│                                                                             │
│  OPTION B — Any Network / Remote Sharing (works anywhere)                  │
│    • Start the app server: npx expo start --tunnel                          │
│    • This generates a public link instead of a local one                   │
│    • Share the QR code or the "exp.host/..." link via text/email           │
│    • Friends install "Expo Go" and scan it from anywhere                   │
│    • Requires your Mac to be running the server while they use it          │
│                                                                             │
│  OPTION C — EAS Build / TestFlight (best — works like a real app)         │
│    This creates a standalone app that doesn't need your Mac running.        │
│    Steps:                                                                   │
│      1. Create a free Expo account at expo.dev                             │
│      2. In Terminal:                                                        │
│           npm install -g eas-cli                                            │
│           eas login                                                         │
│           cd "/Users/mac/Desktop/Claude research app/mobile-app"           │
│           eas build --platform ios --profile preview                        │
│      3. This builds the app in the cloud (~10-15 min, free tier)           │
│      4. You get a TestFlight link to share with up to 100 testers          │
│    • NO Apple Developer account needed for TestFlight internal testing      │
│    • Looks and feels like a real installed app                              │
│                                                                             │
│  OPTION D — App Store (long-term, public release)                          │
│    • Requires Apple Developer account ($99/year at developer.apple.com)    │
│    • Run: eas build --platform ios --profile production                    │
│    • Then: eas submit --platform ios                                        │
│    • Apple reviews it (1-7 days) → goes live on App Store                  │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│  MOBILE APP TROUBLESHOOTING                                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  "Metro bundler fails to start" or strange startup errors                   │
│    → Run: npx expo start --clear                                            │
│                                                                             │
│  "Dependency errors" on npm install                                          │
│    → Run: rm -rf node_modules && npm install                               │
│                                                                             │
│  "Incompatible SDK version" error on phone                                  │
│    → Update Expo Go on your phone to the latest version from the App Store │
│    → Then run: npx expo install --fix (auto-corrects package versions)     │
│    → Then run: npm install                                                  │
│                                                                             │
│  Events not loading on mobile                                                │
│    → Same database as the website — if the website shows events, mobile    │
│      should too. Check your internet connection.                            │
│    → Check if Supabase is paused (see Section 8)                           │
│                                                                             │
│  Biometric login says "no saved session"                                    │
│    → Log in once with your password first. Biometric then re-uses it.      │
│                                                                             │
│  App takes ~30 seconds to analyze a URL                                     │
│    → Normal. The Render backend is waking up from sleep. See Section 8.    │
